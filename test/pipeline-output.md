## 4.1 二叉搜索树的定义 (Binary Search Tree Definition)

二叉搜索树（Binary Search Tree, BST）是一种满足以下性质的二叉树（binary tree）：对于树中的任意节点 X，其左子树（left subtree）中所有节点的键值（key）均小于 X 的键值，其右子树（right subtree）中所有节点的键值均大于 X 的键值。

> **定义 (Definition) — Binary Search Tree（二叉搜索树）**
>
> A Binary Search Tree is a binary tree where each node has a key, and the tree is organized such that the left subtree contains only nodes with keys less than the node's key, and the right subtree contains only nodes with keys greater than the node's key.
>
> 二叉搜索树是一种二叉树，其中每个节点含有一个键值，且整棵树满足如下组织方式：左子树仅包含键值小于该节点键值的节点，右子树仅包含键值大于该节点键值的节点。

这一性质使得高效查找成为可能：在每一个节点处，将目标键值与当前节点的键值进行比较，即可确定应当向左子树还是右子树继续搜索，从而在每一步中排除剩余树的一半。

## 4.2 查找操作 (Search Operation)

二叉搜索树中的查找操作（search operation）遵循一个简单的递归算法（recursive algorithm）。从根节点（root）开始，将目标值 X 与当前节点的键值进行比较。若 X 较小，则搜索左子树；若 X 较大，则搜索右子树；若二者相等，则表示目标元素已找到。

二叉搜索树查找操作的时间复杂度（time complexity）为 $O(h)$，其中 $h$ 为树的高度（height）。在最优情况下（即树是平衡的），$h = O(\log n)$。在最坏情况下（即树退化为类似链表（linked list）的形态），$h = O(n)$。

```c
TreeNode* search(TreeNode* root, int key) {
    if (root == NULL || root->key == key)
        return root;
    if (key < root->key)
        return search(root->left, key);
    return search(root->right, key);
}
```

*代码：二叉搜索树中的递归查找（Recursive search in a BST）*

> **注 (Note)**：递归实现虽然简洁优雅，但在树深度极大时可能导致栈溢出（stack overflow）。迭代（iterative）版本可以避免此问题。

## 4.3 插入操作 (Insertion Operation)

要向二叉搜索树中插入一个新键值，我们首先沿树搜索该键值，直至到达一个空指针（NULL pointer）位置，然后在该位置插入新的节点。该算法自然地保持了二叉搜索树的性质。

```c
TreeNode* insert(TreeNode* root, int key) {
    if (root == NULL) {
        TreeNode* node = malloc(sizeof(TreeNode));
        node->key = key;
        node->left = node->right = NULL;
        return node;
    }
    if (key < root->key)
        root->left = insert(root->left, key);
    else if (key > root->key)
        root->right = insert(root->right, key);
    return root;
}
```

*代码：向二叉搜索树中插入节点（Insertion into a BST）*

> **定理 (Theorem) — 插入操作保持 BST 性质 (Insertion BST Property)**
>
> Insertion into a BST always creates a new leaf node and preserves the BST property. The time complexity is $O(h)$ where $h$ is the height of the tree.
>
> 向二叉搜索树中插入节点总是在叶节点处创建新节点，并保持二叉搜索树的性质。其时间复杂度为 $O(h)$，其中 $h$ 为树的高度。
>
> **证明 (Proof)**：插入算法沿与查找该键值相同的路径行进。当到达空指针位置时，即已找到正确的叶节点位置。由于所有比较操作均保持了二叉搜索树的不变式（BST invariant），因此得到的树仍是一棵合法的二叉搜索树。
>
> The insertion algorithm follows the same path as a search for the key. When it reaches NULL, it has found the correct leaf position. Since all comparisons maintain the BST invariant, the resulting tree is still a valid BST.

## 4.4 删除操作 (Deletion Operation)

删除（deletion）是三种基本操作中最复杂的一种。根据被删除节点的情况，分为以下三种情形（cases）：

1. 节点为叶节点（leaf node）：直接删除该节点。
2. 节点仅有一个子节点：用其子节点替换该节点。
3. 节点有两个子节点：找到其中序后继（inorder successor，即右子树中键值最小的节点），将其键值复制到待删除节点中，然后删除该后继节点。

**示例 (Example) — 从 BST 中删除节点 15（Deleting node 15 from a BST）**

以含有两个子节点的节点为例，展示三种情形下的删除逻辑。

1. Step 1: 找到节点 15（含两个子节点：12 和 20）—— Find node 15 (two children: 12 and 20)
2. Step 2: 找到其中序后继 = 17（右子树中的最小值）—— Find inorder successor = 17 (smallest in right subtree)
3. Step 3: 将 17 复制到该节点的位置上—— Copy 17 to the node's position
4. Step 4: 删除原先包含 17 的节点（该节点至多有一个子节点）—— Delete the original node containing 17 (it has at most one child)

| 情形（Case） | 条件（Condition） | 时间复杂度（Time Complexity） |
|---|---|---|
| 叶节点（Leaf） | 无子节点（No children） | $O(h)$ |
| 一个子节点（One child） | 其中一个子树为空（One subtree NULL） | $O(h)$ |
| 两个子节点（Two children） | 两个子树均存在（Both subtrees exist） | $O(h)$ |

## 4.5 遍历 (Traversal)

二叉搜索树支持三种标准的深度优先遍历（depth-first traversal）方式：

- **中序遍历（Inorder）**：左子树 → 节点 → 右子树。按排序顺序访问各键值。
- **先序遍历（Preorder）**：节点 → 左子树 → 右子树。常用于创建树的副本。
- **后序遍历（Postorder）**：左子树 → 右子树 → 节点。常用于树的销毁。

> **定理 (Theorem) — 中序遍历的有序性 (Inorder Traversal Sorted Order)**
>
> Inorder traversal of a BST visits all keys in sorted (ascending) order.
>
> 对二叉搜索树进行中序遍历，将按排序（升序）顺序访问所有键值。
